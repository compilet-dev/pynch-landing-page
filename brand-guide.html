Colour Theory — Why These Three Work
Relationship
Split-complementary triad. Coral (#FF4D5A, hue ~356°) and Teal (#00D4AA, hue ~164°) are near-complementary. Purple (#7B50C8, hue ~263°) bridges them as the third vertex of a split-comp triangle.
Why it feels balanced
Coral is warm/energetic (action). Teal is cool/trustworthy (success). Purple is prestige/depth (brand authority). Each colour claims a different emotional register.
Coral's role
High-energy, warm-red. Used sparingly for CTAs and urgency. Never used for large backgrounds — only buttons, badges, accents. Too much coral = alarm.
Teal's role
Cool, high-trust. Used for "Pynch wins" moments in the comparison table, teal folder, success states. Signals: this works, it delivers results.
Purple's role
Depth and sophistication. Used for nav CTA gradient, shader background, email form submit. Commands attention without aggression. Pairs with both coral and teal.
Saturation harmony
All three are highly saturated (S > 60%) at similar luminance levels (~50%), so they can coexist without one washing out another — unlike pastels mixed with neons.

Logo Usage

The Pynch wordmark

The Pynch logo is a wordmark set in Righteous — a rounded display typeface that gives the brand a friendly but confident personality. The SVG paths are in pynch-wordmark-mocra.svg. Never set "pynch" in any other typeface for the logo.

✓ Do — Dark background
White wordmark on #0a0a14 — primary dark use
✓ Do — Cream background
Dark wordmark on #FAF8F5 — cream/light use
✗ Don't — Wrong typeface
pynch
PYNCH
Never set wordmark in Plus Jakarta Sans or any other typeface
✗ Don't — Coloured wordmark
Never recolour the wordmark in brand accents — monochrome only
Typeface
Righteous (Google Fonts) — only weight: 400. Nav logo uses height: 24px (mobile) / height: 28px (desktop). Apply filter: brightness(0) invert(1) for dark backgrounds.
Colours allowed
White (#FFFFFF) on dark. Dark ink (#131322) on cream/light. Never coral, teal, or purple on the wordmark itself.
SVG source
pynch-wordmark-mocra.svg — viewBox 0 0 1922 602 (original coordinates × 10 in the web version used here). Created by potrace 1.16.
Minimum size
20px height in nav. Never smaller in production use — glyphs break below this.
Clear space
Maintain at least 1× the logo height as clear space on all sides. Never crowd the wordmark against other elements.
Nav implementation
<img src="pynch-wordmark-mocra.svg" height="24" style="filter:brightness(0) invert(1)"> — on a dark background this renders white. On a light background, remove the filter.

WebGL Shader / Gradient Canvas

The living background

Pynch's hero background is a custom WebGL2 fragment shader — not CSS gradients. It renders 4 brand colours as smooth Perlin-noise-displaced gradient blobs, with a subtle grain overlay. The canvas is fixed behind all content and follows the mouse.

const state = {
  // Brand colour inputs (hex → [r,g,b] 0-1)
  color1: hex3('#7B50C8'),  // purple
  color2: hex3('#00D4AA'),  // teal
  color3: hex3('#9B70E8'),  // purple-light
  color4: hex3('#FF5A45'),  // coral-warm

  // Layout
  colorSize:     1.0,
  colorSpacing:  0.60,
  colorRotation: -0.38,
  colorSpread:   5.40,

  // Warp & zoom
  displacement:  4.20,
  zoom:          0.72,
  spacing:       4.27,
  seed:          Math.random() * 100 - 50,  // random each load

  // Grain
  noiseSize:     0.5,
  noiseIntensity:0.028,

  // Position offsets
  colorOffset:       [-0.7741, -0.2064],
  transformPosition: [-0.28,   -0.44],
};

// Mouse tracking
const BTX = -0.28, BTY = -0.44, MSX = 1.20, MSY = 0.90;
function loop() {
  mx += (tx - mx) * 0.08;  my += (ty - my) * 0.08;
  state.transformPosition = [BTX + (mx - 0.5) * MSX, BTY + (my - 0.5) * MSY];
  state.seed += 0.00045;
  draw(); requestAnimationFrame(loop);
}

/* JS — thumbnail seek + hover-to-play */
const allVideos = document.querySelectorAll('.creator-card video');
const isMobile = 'ontouchstart' in window || navigator.maxTouchPoints > 0;
allVideos.forEach(video => {
  video.addEventListener('loadedmetadata', () => {
    video.currentTime = 0.5;   // thumbnail frame
  }, { once: true });
  const card = video.closest('.creator-card');
  if (isMobile) {
    card.addEventListener('touchstart', e => {
      e.preventDefault();
      playing ? (video.pause(), video.currentTime = 0.5, playing = false)
              : (video.play().catch(() => {}), playing = true);
    }, { passive: false });
  } else {
    card.addEventListener('mouseenter', () => video.play().catch(() => {}));
    card.addEventListener('mouseleave', () => { video.pause(); video.currentTime = 0.5; });
  }
});

/* GSAP ScrollTrigger — card drift on scroll */
gsap.utils.toArray('.creator-card').forEach((card, i) => {
  const direction = i % 2 === 0 ? -1 : 1;   // even = up, odd = down
  gsap.to(card, {
    y: direction * 60 + (i * 20),
    rotation: `+=${direction * 3}`,
    ease: 'none',
    scrollTrigger: {
      trigger: '.card-field',
      start: 'top center',
      end: 'bottom top',
      scrub: 1,
    },
  });
});

Do's and Don'ts

Pynch-specific rules

Rules extracted from what's actually in v16 and what the design system enforces. Not abstract principles — real decisions from the live page.

✓
Do
  • Use Plus Jakarta Sans 800 for all hero headlines and CTAs — it's the primary workhorse
  • Use the WebGL canvas for hero backgrounds. Never replace it with a static CSS gradient for the main landing page
  • Use the two-zone system: cream (#FAF8F5) for light content sections, dark (#0a0a14 / #161616) for rest
  • Always transition dark→grey with the wave SVG path. Never a hard cut
  • Stickers must always be rotated — never flat at 0°. Rotation range: ±4° to ±14°
  • Use coral only for actions and urgency (badges, CTA buttons, coral eyebrows). Not for large fills
  • Use teal for "Pynch wins" moments — comparison highlight, success states, checkmarks
  • Use purple for nav CTA and shader — it signals depth and authority, not urgency
  • Use Geist Mono for stats, labels, badges, code. Never for body copy
  • Use the Righteous wordmark SVG for the logo. Never typeset "pynch" in any other font
  • Apply filter: brightness(0) invert(1) to the wordmark SVG on dark backgrounds
  • Use letter-spacing: -0.03em on h2 headings and -0.04em on h1 — tight tracking is a Pynch signature
  • Use glass cards over the WebGL canvas or images — they need non-opaque backgrounds
  • Use the folder SVG shape (with rounded top-left tab) for major feature sections — it's a brand signature
✕
Don't
  • Never invent a different logo — not a "Y mark", not an icon, not a custom letterform. It's the Righteous wordmark
  • Never use pure white (#FFFFFF) as a section background — Pynch uses cream (#FAF8F5), not white
  • Never use coral for large section backgrounds or folder fills — it reads as alarm/danger at scale
  • Never use more than 3 accent colours on a single page — the system maxes at three
  • Never use Righteous for body text, headings, or UI labels — logo only
  • Never place stickers at 0° rotation — it looks mechanical and kills the brand personality
  • Never use text-30 (rgba white 30%) for body copy — decorative use only (labels, timestamps)
  • Never skip the ScrollSmoother wrapper on pages using GSAP animations — it's required for the horizontal scroll and folder card effects
  • Never recolour the Pynch wordmark in coral, teal, or purple — monochrome only (white or ink)
  • Never use a CSS radial-gradient hero background as a substitute for the WebGL canvas on the main landing page — it will look noticeably different
  • Never use Inter, Instrument Serif, or other non-Pynch fonts — the system is Plus Jakarta Sans + Geist Mono + Righteous only
  • Never set button text lighter than 700 weight — 800 is the minimum for CTA buttons
  • Never skip the grain-overlay SVG — it adds warmth and prevents the WebGL canvas from looking artificially digital
